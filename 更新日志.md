# HMCL-AE 更新日志

## 0.1.0-alpha（开发中，未发布）

> 版本号序列重置：不再延续 0.3.0-beta 的语义化递增，标志架构重写正式进入 Alpha 阶段。首个对外版本号为 `0.1.0`（alpha），由 `ae-release.yml` 手动触发发布时填入。

### 改进

- **AI 出错时会给出更有用的说明，不再原地打转**：以前工具报错常常只有一句「失败了」，AI 拿不到该怎么办的信息，就反复重试甚至想歪招绕过。现在报错统一说清三件事——发生了什么、能不能重试、下一步该干嘛，而且会把 AI 正在找的东西直接给它：装模组版本对不上会列出这个加载器和游戏版本下实际可用的版本，实例名写错会列出真实的实例名单，改存档路径找错会列出该节点下有哪些键。
- **你在确认框里拒绝过的操作，AI 不会再反复弹窗问你**：以前明确点了「否」，AI 换个说法又发起同一个危险操作，同一个确认框能接连弹三次才被系统拦住。现在拒绝一次就记住，本轮对话里 AI 不会再拿同一操作骚扰你。
- **AI 每次对话终于走你配置的代理了**：以前 AI 聊天、拉取模型列表走的是直连，无视启动器里配好的代理——国内网络下经常连不上还查不出原因。现在 AI 的所有联网（对话、模型列表、联网搜索、抓网页、诊断上报）默认都走你的代理，需要账号密码的代理也支持了。

### 修复（数据安全 · 重要）

- **游戏开着时，AI 不会再动正在玩的存档和实例了**：以前游戏正跑着，让 AI 改这个存档的数据、删这个实例、或恢复备份覆盖当前世界，都可能直接执行——轻则存档错乱，重则整个进度被毁且毫无提示。现在这些操作会先检查世界/实例是否正被游戏占用（尊重 Minecraft 的存档锁），占用时直接拒绝并提示你先退出游戏或关掉实例。
- **恢复世界备份中途崩溃，存档不会再「凭空消失」**：恢复过程改成「先把现存档挪到隐藏的 .replaced 文件夹 → 换入备份 → 成功才收尾」，任何一步中断都会留下标记；下次启动那个实例前会自动扫描，发现残留就告诉你原存档在哪、怎么改名找回。备份本身也全程持锁、并排除 session.lock，不会再做出「半新半旧」的坏快照。
- **让 AI 取消下载，是真的取消了**：以前 AI 回你「已取消」，后台线程其实还在偷偷把东西下载装完。现在取消会一路传到底层任务，回执和磁盘状态一致；你按「停止」时，这一轮 AI 派发的后台任务也会一并停掉（并告诉你停了几个）。
- **AI 说「已完成/已在后台执行」时，是真的成功了**：修掉了两类「假成功」——后台任务在线程池不可用时其实立刻就失败了却回报「已开始」；编辑文件时新旧内容相同却回报「已修改」。现在这些都会如实报失败。
- **AI 装模组/删实例走的是启动器本体的同一套逻辑**：mod 的启停/删除/更新收编进了启动器的 mod 管理机制（原生的「回滚到上一版本」因此重新有效），AI 建实例套用你设的默认版本隔离预设，删实例也走本体的删除流程——AI 做完，本体对应的界面能正确反映，不再是两套各说各话。

- **AI 设置页大改版**：侧栏重组为 全局设置 / 模型服务 / 技能 / MCP 服务器 / 网络搜索 / 数据设置 / 高级设置 / 帮助 / 关于；「AI 能力与行为」并入技能页（作为能力权限入口），「我的数据与安全」并入数据设置，「高级与开发者」进新的高级设置页；删掉了从未起作用的「回复语言」和「AI 标题命名」两个死设置。
- **审批模式合并为 Auto**：不再有 安全/询问/YOLO 三档选择。Auto 模式日常操作自动放行、高危操作弹确认；后台挂机（无人值守续跑）期间高危操作直接禁止执行，而不是弹一个没人回答的确认框。
- **内置技能不再出现在技能列表**：技能页现在只显示你自建的技能；内置技能改为程序内置资源，不再以可编辑的 skill.md 文件形式存在，避免误当成自定义技能改坏。
- **MCP 服务器改为 JSON 编辑器**：直接编辑完整 JSON 配置（校验通过才能保存），并新增 args 参数数组与 env 环境变量支持——以前只能填一行命令字符串按空格切分。
- **网络搜索 API Key 按服务商分开保存**：Tavily / SearXNG / 博查 / 智谱各存各的 key，切换服务商不再互相覆盖。
- **AI 的任务清单更守规矩**：清单改为任务粒度（「安装 Mods → 验证结果」，不再一个模组一行）；AI 完成一项会回去勾选，若把清单晾着不管或悄悄丢弃重开，系统会提醒它回来更新。
- **AI 界面全面对齐 HMCL 原生风格**：聊天相关界面做了一轮原生化重构——按钮、菜单、弹层、抽屉、折叠卡全部换成启动器本体同款控件与配色，自定义样式砍掉约一半（AI 专属样式类从 105 个减到 45 个），深浅色主题下和其它页面终于是一家人。思考卡/工具卡/任务清单统一成同一种折叠标题栏（整行可点、箭头旋转动画，点击高亮不再溢出到奇怪的范围）；消息列表按回合分组、间距更有节奏，气泡去掉描边更清爽；复制/重发/分支/删除那排小按钮改成鼠标悬停到消息上才浮现，平时不再密密麻麻（位置是预留好的，浮现时页面不会跳动）。
- **会话搜索改为原生对话框**：自绘的搜索浮层换成 HMCL 标准对话框——输入即搜、上下键在结果间移动、回车跳转、ESC 关闭，跳转定位后仍有高亮闪烁。
- **会话「⋯」菜单和思考档位菜单原生化**：换成 HMCL 标准弹出菜单，每项带图标；思考档位在当前档打勾，任何主题下每一行文字都清晰可读。
- **思考档位不再讲黑话**：low/medium/high/xhigh/max 显示为「快速/平衡/深入/更深入/极限」（none=不思考），悬停还能看到原始档位名；设置页里同步换用同一套叫法。
- **空聊天页会引导你配置**：还没配置模型服务时，空状态直接给一条提示和「配置模型服务」按钮，不再摆一排点了也发不出去的建议；配好之后建议按钮换成原生描边样式。
- **回复失败不再只有一行灰字**：出错时显示带错误图标和「回复失败」标题的错误卡，下面是带刷新图标的「重试」按钮，一点就重发上一轮。

### 修复

- **装模组不再装错加载器/游戏版本**：以前 AI 装模组时没法把「这是 NeoForge 实例、游戏版本 1.21」这些已经确认的信息传给安装工具，工具就退化成「挑全网发布时间最新的那个版本」——NeoForge 实例装出 Fabric 版就是这么来的。现在加载器、游戏版本、具体版本号会一路传到底。
- **AI 用 shell 命令直接下载模组类文件时会被拦下确认**：检测到「下载动词 + 模组目录或模组站域名」的组合就强制弹确认（即使关了危险操作确认也弹）——这类操作本应走内置安装工具，走 shell 会绕过所有版本匹配检查。
- **思考等级下拉框文字看不清**：全部选项（不只是选中的那个）都有清晰的文字颜色了。
- **思考过程卡与工具卡大小不一、不对齐**：两者的圆角、边框、内边距、标题字号已完全统一。
- **后台任务的「2/10」不再像倒计时**：数字后面标明「运行中」，和聊天里「已完成 N/M」的徽标不再打架。
- **问某个实例，不再答成当前选中的那个**：以前你指名问某个实例（比如刚导入的整合包），实例类工具若没认出参数名，会不声不响地拿「当前选中实例」的信息来回答——AI 被带偏后就开始绕路瞎猜（07-06 配置 Simple Love 的会话就是这么翻车的）。现在约 20 个实例类工具统一了规则：指名的实例不存在就明确报错并列出真实实例名，绝不偷偷换成选中实例；纯查询类工具还能听懂 name/query 等常见参数写法。
- **批量装模组后不再被"延迟回执"刷屏**：一口气装十几个模组时，每个后台任务完成都会单独插一条"（后台任务 #N 已完成）请据此继续"，AI 只能反复回答"这是延迟回执，早已就绪✅"——刷屏又烧 token。现在同一批完成的任务合并成一条回执一次性交给 AI；AI 已经用 check_job / list_jobs 亲眼确认过结果的任务，不再重复通知。
- **AI 设置页改完不显示**：技能启停、重新扫描技能目录、编辑/删除 MCP 服务器、删除记忆等操作，以前改完界面纹丝不动，得切走再切回来才能看到；现在改完立即刷新。
- **改设置不再掐断正在生成的回复**：以前 AI 正在回答时保存任何 AI 设置，进行中的回复会被直接打断成「连接错误」；现在当前回合按旧设置跑完，新设置从下一回合生效。删除正在生成回复的会话也会先正规停掉那条流，不会再把发送键永久卡死。
- **聊天设置抽屉的开关不再失忆**：工具调用显示、流式输出、回车发送、自动滚动四个开关和思考强度，以前改完只在当次生效、重启就弹回去；现在改完即存盘。
- **「字号」设置对 Markdown 终于生效**：以前调字号只影响纯文本，Markdown 消息（标题/粗体/行内代码/代码块/表格）的字号写死在代码里纹丝不动；现在整套排版跟着字号档位缩放。
- **国旗 emoji 不再空白**：🇨🇳 这类国旗以前被拆成两半去找图，永远显示空白；现在成对识别，图片下载失败时也回退成文字而不是留白（失败的图不再被缓存到重启）。
- **设置页改的搜索/OCR 配置，聊天里立即用上**：以前设置页改的是自己那份副本，聊天里的搜索工具还攥着旧配置（换了搜索服务商还在用旧 key）；现在两边共享同一份，改完即生效。
- **大附件、会话保存、数据备份不再卡界面**：这些磁盘读写挪到了后台线程；流式输出的界面刷新也做了合批，长回复时明显更顺。

## v0.3.0-beta（预发布）

### 新增

- **技能不再靠 AI 自觉了：命中就自动喂给它**：以前技能目录只是写在提示词里，AI 想不起来去读就等于没有——弱一点的模型基本从来不读。现在你的消息只要命中技能的触发词（比如说「闪退」「装个光影」「帮我备份存档」），对应的技能手册会整篇自动放进 AI 的上下文，让它照着一步步做；整个对话期间手册保持在场，说「继续」也不会丢。可在 AI 设置 →「AI 能力与行为」→「技能自动匹配」关闭。
- **内置技能从 6 个扩到 13 个**：新增联机进服（含 ping 检测、版本/模组匹配、局域网）、存档管理与备份恢复、改存档数据（背包/传送/游戏模式，NBT 全程先备份）、光影与资源包（会先装 Iris 等前置）、下载与网络问题排查（换源/代理/重试规则）、Java 与内存（版本对照表、内存建议值）、整合包（装/导出/整包更新）7 个场景手册；原有 6 个技能全部补上了触发词。
- **首次使用 AI 会先说明数据外发**：第一次打开 AI 助手时弹一次「隐私与数据说明」——讲清用 AI 时哪些内容会发给你配置的服务提供商（问题、游戏信息、你让它读的文件/日志、剪贴板、截图等），点确定即表示知晓同意；之后可在 AI 设置 →「我的数据与安全」→「隐私与数据说明」随时重看。
- **AI 花费护栏：可设每日上限，防账单爆炸**：AI 设置里新增「每日花费上限（美元）」——按各模型定价估算单日花费，达到上限后暂停发送（提示你调高上限或明天再用），用到约 80% 时先预警；单条回复的估算花费也会显示在气泡下（需给模型填了定价）。0=不限。
- **能看到模型的「思考过程」了**：用带推理的模型（如 DeepSeek-R1）时，它的思考内容会在回答上方以可折叠的「思考过程」卡片显示——生成时展开、回答一开始自动收起，点标题随时展开/收起；重开会话也会保留（默认收起）。目前支持 OpenAI 兼容接口的推理模型（DeepSeek-R1 等）。
- **代码块升级：语言标签 + 一键复制 + 语法高亮**：AI 回复里的代码块顶部会显示语言名和「复制」按钮（点一下整段复制、变「已复制」），代码本身做了轻量高亮——注释、字符串、数字、常见关键字分色显示。
- **全局记忆可以逐条删除了**：设置 →「全局记忆」里每条记忆右侧有删除按钮（确认后删除，不可撤销）；点条目仍可打开记忆目录，直接编辑对应的 markdown 文件。
- **支持导入会话**：新增 `/import` 命令（输入框打 `/` 可补全），把之前「导出全部会话」得到的 JSON 重新导入——只新增、绝不覆盖已有会话，重复导入也不会产生副本。这样导出↔导入就形成了备份闭环。
- **AI 读大文件不再一次性灌爆上下文**：`read` 工具读大文件/日志时，默认只回最后 200 行并注明总共多少行，要看前面用 `startLine` 翻页——以前动辄把整份日志塞进去，几下就把模型的上下文占满、后面全乱。
- **单个工具结果默认封顶约 2 万字**：读文件、抓网页等返回超长时会截断并提示 AI 换更精确的参数重读（设置里「工具结果长度上限」默认 2 万字；想彻底放开设为 0＝不限）。
- **长对话的工具循环会自动省内存**：一次回复里连续调用很多工具、累计内容超过模型窗口时，会把最早的几条工具结果自动折叠成占位符（AI 需要时可重新调用），避免越滚越大直接撞上下文上限报错。

- **输入框终于是多行的了**：随内容自动长高（最多约 8 行，超出内部滚动），Enter 发送、Shift+Enter 换行；关掉「回车发送」后 Enter 换行、Ctrl+Enter 发送。粘贴长日志不再挤成一行。
- **中文输入法不再误发**：打拼音按回车只是上屏，不会把没打完的字直接发出去。
- **每个会话有自己的草稿**：切走再切回来，没发出去的字还在。
- **附件改成独立小卡片**：可以一次附多个文件，文件内容发送时才读取、不再灌进输入框；删掉某个附件只删它自己，绝不清空你打的字。拖拽多个日志文件也都能挂上。

- **AI 做过的操作现在会留在聊天记录里**：每次工具调用（装模组、备份存档等）都作为独立记录存进会话，重开会话还能看到当时执行了什么、成功还是失败——以前回复一结束这些卡片就全消失了。
- 消息带上了回合、时间和所用模型的信息（为后续界面升级铺路），旧会话文件照常兼容。

### 新增（会话列表）

- **会话终于能重命名了**：会话行右侧改成「⋯」菜单，里面有重命名 / 置顶 / 删除——以前自动起的标题不满意只能删掉整个会话。
- 会话可以置顶，置顶的排最前面；每行还显示最近聊天时间（今天显示几点、昨天、几月几日），删除确认也会带上会话名。

### 修复

- **更抗“提示注入”**：AI 从网页、文件、日志、崩溃报告等外部来源读到的内容，现在会被明确当作“数据”而非“指令”——里面夹带的“忽略上面的话 / 删除存档 / 交出密钥”之类不会被照做；网页抓取结果还加了不可信内容边界标记。涉及删除等不可逆操作若依据的是外部内容，会先跟你确认。
- **AI 卡住时会自动收手**：一次回复里连续多次工具调用失败、没进展时，会自动停止盲目重试，改用已有信息作答或说明卡在哪、需要你提供什么——不再一直空转到轮次上限。
- **AI 记忆存前自动脱敏 + 去重**：AI 往长期记忆里写内容时，会先抹掉 API key / token / 密码这类密钥（存成 `[REDACTED]`），不让密钥落进明文记忆文件；内容完全相同的记忆也不再重复保存。
- **模型 API 调用现在会写进应用日志**：每次请求/响应的轮次、耗时、token 用量，以及瞬时重试、请求失败都会记一行 `[AI]` 日志——出问题时排查有据可查，不再只有会话记录。
- **「工具结果长度上限」的 0 恢复为「不限」**：之前一版把 0 悄悄改成了「自动 2 万字」，改动了这个设置的原意。现在 0 就是不截断（你显式的选择），默认值改为明确的 2 万字（防单次超大读取撑爆上下文）。想彻底不截断把它拉到 0 即可。
- **会话正在生成回复时，侧栏该会话的图标改为 JFoenix 转圈动画**（之前临时用的是一个圆点符号 ●）。
- **一个会话正在回复时切到别的会话，不再误伤**：以前在另一个会话点发送，会把前一个会话正在生成的回复掐断，或者把你新打的话无声吞掉。现在「停止」按钮只在正在生成的那个会话里出现；切到别的会话就是正常的发送按钮，此时若发送会提示「另一个会话正在生成回复，切回那个会话可以停止它」，不再误杀、也不再静默吞。侧栏里正在生成的会话会显示「● 正在生成…」。
- **MCP 工具从「列得出、用不了」变成真能用**：以前接入 MCP 服务器后，那些工具只是摆设——AI 一调用只会收到一句「运行时执行待接线」的占位符。现在会真正通过 MCP 协议执行并把结果回给 AI；工具的参数也会正确带上（不再只有一个通用的 query），AI 能按 MCP 服务器声明的参数正确调用。
- 斜杠命令补全弹出时默认选中第一项，回车直接生效；命令打完整了按回车就是执行，不再被无声吞掉。上下键循环选择。
- 反复点「新建对话」不再堆出一排空会话——当前会话本来就是空的就直接聚焦输入框。
- API Key 保存时会清掉粘贴混进来的换行和空格，不再莫名 401。
- 分支会话的标题后缀从「✦」改成「（分支）」，一眼能看懂。
- 后台任务完成后的自动续跑提示不再伪装成"你发的消息"——改成居中的浅色事件条，悬停可看完整内容，也不会再被复制出一串内部拼接文本（这是之前反馈的气泡错乱问题的根治）。
- 后台任务完成时如果你正好在打字，草稿不会再被顶掉发出去了；崩溃报告转给 AI 时同样不再动输入框。
- 你切到别的会话时完成的后台任务，现在会等你切回原会话时再自动续跑，不再被悄悄丢掉。
- 「说一段→调工具→再说一段」的回复完成瞬间不再出现前一段文字重复闪现；出错时也一样。
- 对 /compact 压缩摘要那条消息点"重新生成"不再一键清空整个会话，会提示它没有对应的提问。
- 编辑消息的确认/取消按钮在有回复正在生成时会先拦一下，不再能把正在写入的历史截断。
- 回复生成中切换 Markdown/用量等显示开关不会再打断正在进行的提问面板和流式气泡了（改动会在下次渲染生效）。
- 删光所有会话后在顶栏切换模型不再崩溃。

- 「技能与工具」里的技能开关之前是个摆设——关了 AI 照用、重启还原。现在真正生效并跨重启保留。
- 「AI 标题命名」「自动崩溃分析」两个开关改完即存盘，不再重启就弹回去。
- 改模型 ID 时如果它正是默认模型，默认模型会跟着改名——以前改完照样拿旧 ID 发请求，一直 404 还查不出原因。
- 「导出全部会话」挪到后台线程执行且兜住所有异常：会话文件损坏时会提示导出失败，而不是直接把启动器弄崩；大历史也不再卡住界面。
- 添加 MCP 服务器后点取消，不再残留一个幽灵空配置。

- 会话记录文件万一损坏（断电、被杀软占用等），不会再被启动时静默覆盖清空——现在会把坏文件原样保留成 `.corrupt-时间` 副本并在聊天里告诉你放在哪，历史随时可找回。
- AI 设置文件（所有服务商配置和 API Key）改成先写临时文件再原子替换，中途崩溃不再可能把整份配置打回默认。
- 上下文窗口设小之后，超出的老消息只是不发给模型了，不再从磁盘上被永久删掉——以前给本地小模型配 8K 窗口，会把每个会话的历史无提示地砍到只剩几千字且找不回来。
- 恢复世界备份的过程改成「先把现存档整体改名挪到一边→换入备份→确认成功才删」，任何一步失败都自动把原存档还原；游戏正开着存档时会直接拒绝并提示，不再删到一半。
- 保存会话/设置失败不再完全无声——现在会记进日志，方便排查"我改的东西怎么没了"。
- 修了下拉弹窗在「上下都放不下」时一律向下撑出屏幕的边角问题，现在会挑截断少的一侧。

- 危险命令拦截把 Windows 本地写法全补上了：`rd /s /q`、`del /s`、PowerShell 的 `ri`/`rd`/`del`/`erase -Recurse` 以及 `-rec` 这类缩写，以前这些一个都拦不住，YOLO 模式下会不带确认直接执行。
- PowerShell 编码命令的拦截补齐了 `-EncodedCo`、`-EncodedComman` 这类合法前缀缩写（PowerShell 本来就认它们），不能再靠少打几个字母绕过检查。
- 整数参数不再莫名带上 `.0`——以前让 AI 把渲染距离改成 12，写进游戏配置的却是 `12.0`，游戏直接当没看见。
- 你让 AI 复制的文本如果本身长得像 `text=xxx`，不会再被错当参数前缀剥掉一截。
- 世界类工具（恢复备份、删除世界等）如果只收到了次要参数（比如只有 backupId），不会再把它错当成世界名去找，报出「找不到叫 true 的世界」这种误导 AI 空转的错误。
- shell 工具缺参数时的提示改成和声明一致的参数名，不再教模型用错键。

- 点停止后，中断标记现在立刻写进会话，不会再和你紧接着发的下一句消息抢位置（之前偶发顺序错乱、甚至丢掉）。
- 在工具调用中间或自动重试期间点停止，说了一半的话也能保留了——之前只有普通流式阶段停下才保得住。
- 剥离模型泄漏的工具标记时不再误伤正常内容：以前回复里只要出现 `x < |y|` 这类数学式或 `<|` 管道写法，从那里到结尾会被整段吞掉。
- 「说一段→调工具→再说一段」的回复存进历史时，段落之间补上了空行，不再粘成一坨；工具轮数达到上限时的提示也不再被吞。
- 后台任务的进度轮询（check_job）不再被「重复调用拦截」误杀——之前问到第三次就被强行掐断还谎称结果没变。

## v0.2.1-beta（预发布）

> 这一版主要在收拾 AI 助手的脾气：少绕圈、少乱试、少把界面弄花。也顺手改了设置和聊天的不少细节。

### 新增

- **提供商一键预设**：新建 / 编辑配置时顶部多了一个下拉，挑 DeepSeek、智谱、通义、硅基流动、Kimi、豆包这些常用服务商，地址和协议自动填好，旁边还附了直达控制台去拿 API Key 的链接。
- **导出全部会话**：数据管理里新增一项，可以把所有对话一次导成 Markdown / JSON / 纯文本。
- **后台任务上拉框**：输入框上方那条后台任务栏收成了可折叠的小条，平时只显示个数，点开按类目列出，想取消哪个点一下就行。

### 改进

- 全局设置重新分了四类：对话与模型、AI 能力与行为、我的数据与安全、高级与开发者。前两页只放日常要调的，危险开关都挪进了末尾折叠区。
- 聊天里那几个跟输入有关的开关（流式、回车发送、自动滚动）搬进了「聊天设置」抽屉，全局里不再重复出现。
- 聊天设置那个侧栏换了层皮：加了投影和圆角，关掉按钮换回原生圆形那个，看着不那么像糊上去的。
- 编辑消息不再把原话和下面的回复一起删，改成在原气泡里就地改，配取消和确定两个键。
- AI 说完一回合，下面的复制 / 重发 / 分支 / 删除几个键马上就出来，不用再切一下会话才显。
- 提示词瘦了身，常驻那部分稳定下来方便缓存，整体每轮少了约三分之一。
- 名字、标题、关于页正式换成了 HMCL-AE（上游署名照留，正版登录等自有凭据再开）。

### 修复

- 选单选 / 多选常常点歪到隔壁项——JFoenix 那个涟漪框比字大太多，收到了跟控件差不多大。
- 思考等级那个弹窗原来撑得老宽，缩成紧凑几行。
- 输入框正上方有一块看不见却能挡住点击的地方，清掉了。
- 数字 emoji（0️⃣ 这类）画不出来，现在能正常显示了（得开着彩色 emoji）。
- 编辑提供商时不会把原来的配置吃掉了，之前一存就丢每个模型的别名和计费。
- 从旧版迁过来的 API Key 不会被二次编码弄坏。
- Anthropic 不再无视填的接口地址。
- 点停止之后，模型这回合说了一半的话会留下来了——之前直接丢，导致下一句「继续」它全忘了。
- 创建离线账号老报「username is required」：模型把名字放到了 query 里，工具只认 username。现在常见几个参数名都接，名字写成「username=xxx」也认。
- 顺手把这套兜底铺到了十七个工具上——加账号、备份世界、装数据包、设游戏选项、转移物品这些，模型把参数名写错或写成「名=值」都不会再卡死。
- 工具老报错时不再让它一遍遍空转：改成了出错过先找原因、能修就修、修不了就停下来问你，不准拿读写 shell 去手搓替代专用工具。
- 个别模型（deepseek 流式）会把工具调用当成 `<｜…｜>` 这种文本吐进聊天，现在剥掉了；整条都是这种就提示换个能函数调用的模型。
- 崩溃报告窗的「让 AI 诊断」在主窗口关掉时点会崩，现在不崩了，改成把日志塞剪贴板、提示你打开 HMCL-AE 粘过去。
- web_fetch 拒绝往内网和云元数据地址跑，重定向每一跳都重新校验。
- 会话存盘改成原子写，半路断电不会把全部对话写没。
- 恢复世界备份改成先搁临时副本再换进去，不会中途把存档搞坏。
- 危险命令那个二次确认门有个参数绕过的口子，补上了。
- 装模组会按你选的加载器挑版本，不再往 Fabric 实例塞 Forge 的 jar。
- 数据管理里那个不存在的「知识库文件」删了，「知识库 / 技能目录」也改回了「技能目录」。

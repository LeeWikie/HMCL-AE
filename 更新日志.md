# HMCL-AE 更新日志

> 本文件持续记录改动,新内容追加在最上方。

## 2026-06-30

### 界面 / 交互
- 进一步收小 ask 选择题(单选/复选)的点击命中体积:原 JFoenix rippler 框比可见圆点大很多,常误点到相邻选项;现把内边距收到接近可见控件大小。
- 全局设置重新整理为 **4 大类**:对话与模型、AI 能力与行为、我的数据与安全、高级与开发者(危险/进阶项收进折叠的「高级与开发者」)。
- 聊天行为设置(流式输出 / 回车发送 / 自动滚动)统一收进「聊天设置」抽屉的「交互」区,不再与全局设置重复。
- 「聊天设置」抽屉改为 Material 风格:加投影、内侧圆角、原生圆形关闭按钮(内部控件本就原生)。
- **编辑消息**改为原地内联编辑:点编辑把气泡变成输入框 + 取消 / 确定,不再删除该消息及其下方回复。
- AI 回复生成完毕后,消息下方操作图标(复制 / 编辑 / 重发 / 分支 / 删除)**立即显示**,无需切换会话。
- 思考等级选择弹窗改为紧凑样式。
- 后台任务改为**可折叠上拉框**:折叠时仅显示「后台任务 + 运行数」,展开按分类显示「运行/总数」+ 取消。
- 修复选择框点击错位:弹出定位回归 + 修正 JFoenix 单选/复选框命中区整体偏移(影响全局)。
- 修复聊天输入栏上方一处隐形区域遮挡对话点击的问题。
- 数字序号等 keycap 彩色 emoji(0️⃣–9️⃣ / #️⃣ / *️⃣)现可正常渲染(需开启「彩色 Emoji」)。

### 模型 / 提供商
- 新增/编辑提供商对话框顶部新增「**快速预设**」:置顶 3 种协议 + 常用服务商(DeepSeek / 智谱 / 通义 / 硅基流动 / Kimi / 豆包 / 百度 等,国内直连优先);选中自动填入接口地址 + 协议,并在密钥框下方提供「获取 API Key」直达对应控制台。
- 编辑提供商 / 模型对话框会正确**填入现有配置**供修改。
- 测试连通性对话框改为原生可折叠的提供商夹(全选三态:全选 / 半选 / 未选)+ 合理行高;API Key 默认掩码 + 眼睛切换显示。
- 修复:保存时丢失每个模型的配置(别名 / 计费 / 能力);迁移旧数据时 API Key 被二次编码损坏。
- Anthropic 协议遵循自定义接口地址。

### 对话 / 智能体
- 修复创建离线账号常失败:`add_offline_account` 现接受弱模型常用的别名参数(name/query/input 等)并剥离 `username=` 前缀,不再因模型把名字放错参数名而反复报「username is required」。
- 修复崩溃报告窗口「让 AI 诊断」在启动器主窗口已关闭时点击会崩溃(空指针):现做判空,主窗口不可用时把崩溃日志复制到剪贴板并提示用户打开 HMCL-AE 粘贴。
- 改进工具出错时的处理:强化系统准则——出错先诊断、可修则修、瞬时错重试一次、修不了就停下来用 `ask` 问用户;**禁止反复瞎试或用通用工具(glob/读写/shell)手搓去替代专用工具**(如手动改文件加账号)。轮数上限保留为最后兜底,并在触顶时给出清晰提示而非空消息。
- 修复部分模型(如 deepseek 流式)把工具调用以 `<｜…｜>` 文本形式输出、泄漏进聊天界面的问题:现自动剥离这类残留标记;若整条回复都是无法解析的工具调用,提示用户更换支持函数调用的模型。

### 数据 / 安全
- 修复点「停止」后丢失上下文:此前中断会**丢弃**模型本回合已产出的内容,留下两条连续用户消息、模型忘记刚做了什么;现在保留已产出的部分(标注「本回合已被用户中断」),续接对话上下文连贯。
- 数据管理移除名不副实的「知识库文件」项(实为技能目录),并把「知识库 / 技能目录」更名为「技能目录」。
- 「导出为文件」改为「**导出全部会话**」,可选 Markdown / JSON / 纯文本。
- 修复危险命令二次确认门可被特定参数绕过的问题(现扫描全部参数别名,任一危险即拦截)。
- web_fetch 增加 SSRF 防护(拒绝内网 / 回环 / 云元数据地址,逐跳校验重定向)。
- 会话存储改为原子写入,避免崩溃导致全部对话丢失。
- 世界备份「恢复」改为先暂存快照再替换,杜绝恢复过程损坏存档。
- 安装 mod 按所选加载器筛选版本;请求重试的状态分类修正;后台任务取消竞态修正。

### 性能
- 系统提示词缓存友好化,并将场景说明迁入按需加载的技能,单轮提示词体积下降约 1/3。

### 品牌
- 应用名称 / 标题 / 关于页改为 **HMCL-AE**(保留上游 huangyuhui 的版权与开源署名;微软正版登录待自有 Azure 凭据后开启)。

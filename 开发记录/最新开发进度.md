## 〇·2026-07-11 收尾批：定价/成本估算体系归档（决策记档）

**决策**：定价/成本估算体系**正式归档（archived/shelved）**——这是既定决策，不是"暂时隐藏"。UI 全部隐藏，底层代码 + 数据 + `model-library.json` 里的 pricing 全部保留，翻一个 flag 即可整体复活。

- **为什么归档**：
  - HMCL-AE 是 BYOK / 极客定位，成本估算不是重点；
  - 各家 pricing 数据准确性、维护成本都高（models.dev 也可能滞后），显示可能不准的价格反而误导用户；
  - 与其留一套半可信的成本显示，不如整体搁置、需要时再复活。
- **归档范围（全隐藏，均已被 gate）**：模型编辑器的「定价」折叠区、聊天设置的「显示成本」开关、回复页脚的成本后缀（`AIMainPage.formatUsage`）。单一开关 `AIMainPage.PRICING_UI_ENABLED=false`（`AISettingsPage.PRICING_UI_ENABLED` 镜像跟随），`AIMainPageStaticsTest` 断言两开关恒一致且为 false。
- **保留可复活**：价格字段的构建/自动填充/保存、`estimateCost`/`computeCost`、`AiModelEntry` 价格字段、`ModelInfo.hasPricing/computeCost`、`model-library.json` 的 pricing 数据（本会话模型库扩充时从 models.dev 抓取的 1000+ 条）**全部不删**——归档即可复活。
- **怎么复活**：把 `AIMainPage.PRICING_UI_ENABLED` 翻成 `true`（AISettingsPage 自动跟随），模型编辑器定价区、成本开关、页脚成本立即恢复，数据仍在库里，无需重新采集。
- 本批同时落地 OCR 工具热绑（对称 web/shell/nbt，翻开关即时注册/注销 `ocr_image` 无需重启）、思考 pill 显示 effective 推理强度、composer 四弹窗开合动画统一 + 左右自适应定位等 UI 打磨（见 git 提交）。

## 〇、2026-07-11 凌晨：窗口抢焦点根因 + Claude Code 式审批模式 + 通宵全局审计（11 commit 全落地）

**触发**：用户反馈主窗口"时不时自发缩小到固定尺寸 + 强制抢占系统焦点"，排查中同时发现另一个进程（用户自建的 vibe-bench 基准测试）在沙盒副本上频繁触发同一个根因，两条线索共同坐实：**`ManagedProcess.stop()` 硬杀进程后，`ExitWaiter` 的退出码/日志启发式判断把"主动停止"和"真崩溃"混为一谈**——任何一次停止游戏（原生按钮、AI 工具、取消启动、退出清理，共 4 个调用点）都有竞态概率被误判为 `APPLICATION_ERROR`，进而触发默认开启的自动崩溃诊断，无条件 `Controllers.getStage().show()+toFront()`。commit `0c709aaf1`：给 `ManagedProcess` 加 `stoppedIntentionally` 标记，`ExitWaiter` 识别到时归类为既有的 `INTERRUPTED`（未新增枚举值，全仓核查 `ExitType` 消费点后确认安全）；`GameCrashWindow.submitToAiAssistant` 新增 `shouldBringMainWindowToFront(interactive)` 纯函数门控，自动诊断路径不再抢焦点，交互式点击诊断按钮的路径不变。举一反三审计确认全仓再无同类"自动事件无条件抢占前台"模式。

**composer 空态高度 + 审批模式恢复三态**（commit `212c6fad1`）：用户猜测 composer 工具栏里的模型选择器把高度撑起来了——证实：沿用的是 HMCL 原生 `LineComponent` 的 **Java 硬编码 48px 最小行高**（`setMinHeight(48)` 写在构造函数里，CSS 从未真正生效在这个地板上），换成紧凑 `JFXButton` 后卡片从理论 ~133px 实测降到 59px。同批恢复了此前合并掉的审批模式三态（Auto/Ask/yolo），无人值守下危险操作硬阻断这条安全不变式原样保留。

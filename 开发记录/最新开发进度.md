## 〇·前批、2026-07-09：真相修复 + 工具/技能架构调整（大批量落地）

由真实用户实测反馈（"效果很差"）+ 会话尸检（Simple Love 整合包会话六类失败模式）+ 26 条竞品借鉴清单 + 真实诊断 trace 三方交叉验证产出的完整计划，本次会话**全部 25 项实施完成**，`HMCLAI:test` 188/188 绿，`HMCL`/`HMCLAI` 两模块编译全绿。

**一、Trace 层**：修复 `LangChain4jChatAdapter.java` 游离 NUL 字节（已被并行任务修复，本会话核实）；补齐 #44 API 日志的 model/status 字段。

**二、Agent 运行时护栏**：job 唤醒提示词重写（禁止 sleep+check_job 轮询）；通用签名式卡死检测（滑动窗口 3/4/5 级软警告、6 次硬停止，取代原 WAIT_STREAK 特例）；NO_PROGRESS 补第三态（wait-no-op 不再误判为进展）；length 截断防止工具调用带脏参数执行；坏 JSON 参数回执自带 schema；轮次上限软着陆 `forceFinishTurn`（模型自己收尾而非罐头文案）；工具输出超限落盘+指针；verify-on-stop 证据门（写后不验证不能收工）。

**三、工具域合并**：~64 个 Minecraft 业务工具收敛为 6 个域工具——`instance`（本地配置+内容管理，34 action，新增 `set_isolation` 补齐实例隔离切换缺口）、`game`（运行时状态：list 含运行标记 + launch + stop，新增 `LauncherHelper` 按实例 id 跟踪进程，`game.stop` 从"完全没有底层支撑"变为真正可用）、`search`（Modrinth+CurseForge 默认并发查询合并，不再让模型自己猜源头死磕）、`account`、`nbt`（默认隐藏）、`job`。联动改造 `ToolSpec.getPermission(Map)`、`CriticalOperations`、`LONG_TASK_ACTIONS`、`AiToolPermissionStore` 四处 action 级判定。新增 `list_local_ip` 工具补上 shell 关闭后唯一的真实缺口（局域网联机取本机 IP）。

**四、技能系统改造**：`SkillLoader` 支持二级目录（`<domain>/<action>/SKILL.md`）+ `requires:` frontmatter；新增 `SkillIndex`（手写 BM25，CJK 双字符 bigram 分词）作为 Layer 2 模糊检索，与 `SkillMatcher`（Layer 1 精确触发词）分层合并；`MAX_ACTIVE_SKILLS` 3→12，`SKILL_BODY_MAX_CHARS` 6000→1000，`SKILL_MATCH_LIMIT` 2→6；新增 `load_skill` 工具（技能调用不再借用 `read`，UI/trace 语义正确）；用 workflow 并行拆分原 13 个场景技能为 18 个操作级技能 + 13 个 ~800 字编排索引（含对抗性复核，修正 10 处残留旧工具名/超长问题）。

**五、其余架构改动**：`AiPromptBuilder` 拆分 `buildStablePrefix()`（人设/工具指南/纪律/技能索引/静态机器信息，进程级不变）+ `buildVolatileSuffix()`（策略/语言/记忆/技能正文/运行时上下文），`ChatAgent.buildMessages()` 把易变部分挂到当轮用户消息的 wire-only 副本（`<turn-context>` 标签），不再拼进 system 消息，为 prefix-hash 缓存铺路；新增 3 条测试锁定 prefix 字节稳定性 + 软上限。DISCIPLINE 新增 11-14 四条规则（长尾事实溯源、不炒冷饭建议、该问必须问、交付内容对齐承诺）。mod 开关文案 4 处修正（不再教 shell 改后缀）。Shell 默认关闭（挪进"高级与开发者"+ 二次确认弹窗）。全局记忆 `isMemoryEnabled()`/`isAutoRecallMemory()` 强制 false（"待开发"，设置页两行标注禁用态），NBT 默认关闭。

**六、UI bug 修复（4 个，均已编译验证，未做真机视觉验证）**：后台任务面板高度锁死（`animateJobsList` 不再动画 `prefHeightProperty`）；ReasoningCard 缺 `setMaxWidth(480)`；对话框换行失败（EULA/`InputDialogPane`/`JavaDownloadDialog`/`AddAuthlibInjectorServerPane` 四处补 `setWrapText`+限宽，未动 `JFXDialogLayout` 本体降低风险）；**多轮回复合并成单一气泡**——根因是持久化粒度是"整个 turn"而非"每个 segment"，新增 `LlmStreamCallback.onSegmentComplete` 在每个 cycle 收尾（而非每次工具调用）时回调一次，`ChatAgent` 按 segment 各自落盘为独立消息（reasoning 按增量分配，不重复不丢失）；同时 `LangChain4jChatAdapter.convertMessages` 补上把连续 assistant 消息合并回一条再发给模型（否则下一轮请求对 Anthropic 等强制轮替校验的 provider 会报错）。

**已知取舍/待用户确认**：`backup_world` 旧工具随 instance 域合并直接砍掉（未单独跟用户确认，系统提示词早已引导用 `create_world_backup`）；`list_servers`/`ping_server` 按计划整体砍掉，服务器信息只能靠 NBT 读 `servers.dat` 或用户口述；四个 UI bug 修复只做了编译验证，**建议真机场景复现确认**（尤其气泡合并这个改动面最大）。

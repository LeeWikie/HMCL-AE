## 3. 关键设计决策与理由

- **CLI 测试台的价值定位**：无界面自动化 harness 用于弱模型行为验证与 headless 真 bug 捕获——实际抓到 3 个真 bug。ask 不 auto-answer 而靠多轮 session。
- **子代理并行的硬边界**：改同一文件不能并行（互相覆盖破坏编译），只读分析可海量并行。写操作按「文件不重叠」分组、不编译不提交、主循环集中整合。
- **进度总线解耦**：工具侧只 publish、UI 侧只订阅，不改 Tool 接口。
- **备份引擎诚实降级**：先做版本化全量复制+保留 N，明确标注非增量/git，不假装。
- **安全分层**：普通危险确认 + 红色高危二次确认（YOLO 也触发）；危险命令永远硬拦走 confirmHandler，唯一逃生口是 dangerouslySkipPermissions。
- **弱模型鲁棒性为硬约束**：内置默认模型是薅免费额度的弱模型，parseInt 容错、简单 schema、强工具纪律、透明重试都服务于此。
- **额度管理教训**：18 代理海量扇出瞬间打爆额度零产出 → 改 solo 小步编译提交；后期用户明确「固定月资、快重置就该烧满」时才重新放开大扇出。
- **「核查待办 > 第 N 次重审」**：核查 UltraPlan P0 是否真做，比反复审已修代码更能挖出真遗漏。
- **对外文案铁律**（固化为记忆 `feedback-outward-facing-wording`）：README/release/UI/commit/源码注释只平铺事实，禁自夸、禁标榜「(诚实)」、禁提任何外部项目名。

## 4. 遇到的问题与解决（摘要）

- CLI headless `list_game_versions` NPE → 补 `CacheRepository.setInstance+changeDirectory`。
- CLI headless `list_java` 永久卡死 → 补 `JavaManager.initialize()`。
- Gson 把 JSON number 全解为 Double、"4096.0" 抛异常 → 新增 `parseInt`（Number 优先、容忍 .0）。
- OCR 可选依赖 NoClassDefFoundError 逃逸致 Stop 键卡死 → adapter 改 `catch(Throwable)`。
- 删世界 world 参数 `../` 越界 → normalize + startsWith(saves) 校验。
- AiSettings.load() 空/null 文件 NPE 静默重置全部配置 → 加 `if(data==null) return` + 原子写。
- Stop 按钮不真停、停后仍落库+继续跑工具 → 取消标志贯穿 ChatAgent+adapter。
- 自动续聊无上限可无限烧钱 → 加深度上限（3-5）、真人输入重置。
- 权限页显示幽灵工具且喂空的死 MCP 注册表 → 清空 builtInDescriptors、改读真实注册表。
- 搜索下拉「选 Exa 却偷测 Tavily」、默认 Tavily 国内需代理 → 下拉只列已接入、Test 不静默回退、加 Bocha/Zhipu 国内直连并设默认。
- 我方引入的危险命令门 `containsKey` 链选参被绕过 → 后续回归审计抓到并修（扫全部别名）。

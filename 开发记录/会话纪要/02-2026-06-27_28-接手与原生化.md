## 3. 关键设计决策与理由

- **全面复用 HMCL 原生组件、禁止网页皮肤**：用户反复强调"HTML 只能作为功能蓝图，不能作为视觉皮肤"，凡自定义样式都被要求改回 `AdvancedListBox`/`LineSelectButton`/`ComponentList`/`.card` 等原生件。理由：视觉一致性 + 减少自造轮子（多处发现"用了原生件又自搓 CSS 把它毁掉"的反模式）。
- **AI 设置必须是真正的 Navigator 子页面**，而非 `AIMainPage` 内部 `setVisible` 伪切页——为获得原生进出动画和返回栈语义。
- **工具调用改用 LangChain4j 原生 function calling**：正则 `/tool:xxx` 匹配根本不可靠。
- **"优先现有成熟库"路线**：对标 Cherry Studio 后，用户选择用 commonmark、LangChain4j（tool calling/MCP/Anthropic）等现成方案替代自定义实现，明确放弃 AI 画图/翻译面板/笔记等不适合 MC 启动器的功能。**放弃 RAG 知识库**（由用户另一个项目做成外部 MCP Server 提供）。
- **联网搜索独立于 MCP**：让用户直接填 Key 即用。
- **文件工具保守安全**：默认只读、allowlist 根目录、无任意 shell、写前备份；权限分 Safe/Ask/YOLO。
- **HMCLAI 核心保持 JavaFX-free**：为未来剥离成 AEL（Agent Experience Launcher）的 `ael-ai-engine` 留边界。
- **无人值守 vs 交互式的窗口启动约束**：无人值守长线模式禁止阻塞式启动窗口，只编译验证；日常交互式小改动可正常 `:HMCL:run` 启动。

## 4. 遇到的问题与解决

- **编译/运行崩溃**：`Math.clamp`、`button.close`、LangChain4j JDK http client 缺失、shadow jar minimize 删 SPI——均已定位并修复。
- **持久化 bug 连环**：保存无效/删除不写盘/重复空 profile——根因均是漏调 `save()` 或残留旧字段检查，逐一补齐。
- **`sed` 反复破坏文件**：改为 `git checkout` 恢复后用精确字符串替换或整体重写。
- **MCP 依赖版本困局**：经用户提供正确坐标 `langchain4j-mcp:1.16.2-beta26` 解决。
- **远程仓库操作混乱（用户强烈不满）**：最终回退代码、仓库设为私有并改名 HMCL-AE，立下"以后不碰远程仓库、专心写代码"的约束。
- **无人值守效率问题**：每次启动 HMCL 窗口就卡住任务，通过切换"编译验证不启动窗口"的无人值守模式改善，并区分交互/无人值守两种窗口策略。
- **JFoenix 涟漪时长无法改**：`-jfx-rippler-duration` 不是有效 CSS 属性（一直是空操作）——换原生 `navigation-drawer-item` 后接受原生行为。

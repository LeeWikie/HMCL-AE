# HMCL-AE 0.1.2-alpha

0.1.1-alpha 之后的开发版本,重点扩充了 AI 助手对本地实例的操作能力,并为 AI 改写文件加入了改前自动备份的保护。以下为本版本的变更内容。

## 重要更新

### AI 助手 — 实例操作能力大幅扩充

`instance` 工具新增一批动作,均复用 HMCL 原生实现,权限按动作细分(只读 / 受控写入 / 危险写入):

- 配置:设置实例的 Java 运行时、游戏窗口模式与尺寸、启动前后行为、图形后端与渲染器、每实例启动参数;列出全局游戏设置预设;并修复自动 / 固定内存切换未正确写入、内存值被预设静默继承的问题。
- 内容与存档:模组回滚到归档的旧版本、资源包更新检查(可选下载替换)、世界导出为 zip、世界复制、数据包启停 / 删除、把本地已有文件(模组 / 资源包 / 光影)按类型装进实例对应目录。
- 维护:对应"版本管理"菜单的清理(清日志与崩溃报告 / 重下资源索引 / 清资源 / 清依赖库,后两者需确认且游戏运行时拒绝)、管理 HMCL 的 Java 运行时注册表、生成可在启动器外运行的启动脚本、设置实例图标、管理实例 `schematics/` 下的投影文件。
- `game` 工具的 `launch` 新增测试模式:本次启动强制保留启动器窗口并显示日志,不改动持久化的实例设置。
- 批量:一次安装 / 更新多个模组;实例列表附带检测到的加载器与本地模组数。

### AI 助手 — 改写文件前自动备份

AI 改写既有文本配置文件(各类模组配置、`options.txt` 等)前,会先把原文件备份为同目录下的 `<名>.bak`;备份写不成则拒绝本次修改、不动原文件。此保护在所有情况下生效,包括自动放行与无人值守运行,而不再只在弹出确认框时才发生。新建文件与追加写入不受影响。

## 变更明细

### AI 助手

- 修复 Windows + JDK 21 下 Shell 工具输出、以及非 UTF-8 文件读取的中文乱码 / 崩溃。
- 模组 / 资源包 / 光影的搜索结果补充各条目支持的游戏版本与加载器,便于按实际需要的加载器 / 版本选择。
- `grep` 工具支持上下文行(-B / -A);`web_fetch` 在抽取前对响应体加大小上限,避免大页面卡顿。

### 界面

- AI 界面改用 HMCL 原生所采用的 Material Symbols 图标,导航的未选中 / 选中填充切换与原生一致。
- 工具调用卡片恢复分组摘要(工具名 + 折叠计数)并配图标;代码块复制按钮改为图标式,成功时短暂切为对钩。

### 构建与测试

- 新增 Monocle 离屏运行 TestFX 的 `-PmonocleTest`,UI 测试可真实执行且不弹窗抢焦点。
- 为新增的实例工具、以及文件改写前的强制备份补充单元测试。

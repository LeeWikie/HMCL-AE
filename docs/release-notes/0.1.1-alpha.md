# HMCL-AE 0.1.1-alpha

0.1.0-alpha 之后的修订版本。本版启用微软账户登录,并纳入首发后的一批 AI 助手修复。以下为本版本的变更内容。

## 重要更新

### 账户 — 微软账户登录可用
- 发布构建现内置 HMCL-AE 自建 Azure 应用的 Microsoft OAuth client ID,微软正版账户登录不再被"非官方构建"拦截,登录入口恢复可用。
- 通用"非官方构建,无法保证安全性"提示与 client ID 无关,仍会保留。

### AI 助手 — 危险操作防护
- Shell 工具描述加入显式禁令:不得用于终止 java / javaw 进程。
- 新增对"终止 java / javaw 或启动器自身进程"的危险 / 关键操作判定,避免模型误杀正在运行的游戏或启动器自身。

## 变更明细

### AI 助手 — 工具
- 模组安装前报告依赖状态,避免缺依赖导致的连环崩溃。
- 模组搜索结果透出依赖信息,并澄清 loader 描述不参与检索。
- 实例工具补充资源包 / 光影开关,避免以裸 shell 修改文件后缀。
- Ask 工具在输入非法时保留 JSON 语法错误详情,便于定位。

### AI 助手 — 崩溃诊断与稳定性
- 崩溃诊断补充"日志读取失败 / 进程卡住需终止"的处理分支。
- 危险操作前同步落盘,避免异常退出时会话丢失。
- 内容审核(content filter)拒绝时的错误信息友好化。

### 构建与文档
- README 重写为机械化详细介绍页;新增独立 CHANGELOG。
- 修复 check-codes:中文全角括号改半角(checkTranslations)、补标准版权头(checkstyle)。
- 修复一处偶发失败的单元测试(取消转发测试的跨测试竞态)。

- B6 **记忆内容脱敏**（存前过滤 API Key / 密码 / 邮箱 / token）— ❌ `[net-new/ultraplan]` RememberStore 无任何过滤，prompt 注入可诱导把密钥写进持久 md

- C7 附件内容纳入上下文（发送时读取拼入）— ✅

**D. 隐私与合规（上下文外发）** — 详细审计留到领域 6，此处仅登记
- D1 记忆 opt-in — ✅
- D2 发送前告知「哪些内容会外发给模型」— ❌ `[ultraplan/PIPL]` 每轮把 OS/JVM/磁盘/文件/剪贴板/截图发远程，无同意声明
- D3 tool 日志 / 记忆脱敏 — ❌ `[ultraplan]`（与 B6 同源）

### 结论

#51 之后，上下文窗口管理这块已经**扎实**（预算裁剪、工具封顶、超预算驱逐、自动+手动压缩且是续写式摘要、每模型窗口识别、分页读）。记忆也比多数同类完整（存储 / 检索 / 自动召回 / opt-in / 管理 Tab）。真正的缺口收敛为：

1. **记忆安全**（最该补，net-new + ultraplan）：B6 脱敏 + B7 去重——prompt 注入能把密钥写进持久 md 是实打实的风险。
2. **上下文占用可视化 A10/C7**：让用户看得见「快满了、该压缩了」。
3. **记忆管理 UI 只读 B5**：UI 内应能删/改单条，而不是让用户去翻文件夹。
4. **发送内容透明 / 隐私声明 D2**：合规底座，跨领域，留领域 6 细审。
5. **RAG / 知识库 C6**：可选、重投入，暂不立项。

### 本领域待办（按优先级）

- **P1（安全）**：记忆脱敏 + 去重（RememberStore 存前过滤密钥/邮箱、同标题或高相似去重）→ 已建任务。
- **P2**：上下文占用指示（并入 C7）；记忆管理 UI 加删除/编辑 → 已建任务。
- **P3**：跨会话 @提及(C5)、隐私声明(领域6)、RAG(暂缓)。

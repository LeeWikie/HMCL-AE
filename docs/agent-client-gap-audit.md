## 领域全景 taxonomy 与覆盖性检查（结束标准）

一个像样的 AI Agent 桌面应用，按四层拆分应覆盖的领域：

- **交互层**：① 对话 UI / 交互 · ⑫ 多模态（输入/组合器并入①⑫）
- **Agent 内核层**：② 上下文与记忆 · ③ 工具与能力 · ⑬ 规划/自主/多步执行 · ④ 模型与提供商
- **数据/平台层**：⑤ 会话与历史 · ⑧ 可扩展性 · ⑭ 宿主集成 · （网络/代理并入④）
- **工程/质量层**：⑥ 安全权限合规 · ⑦ 可观测性/日志/成本 · ⑨ 错误处理健壮性 · ⑩ 性能并发 · （QA/测试并入③⑦）
- **产品化层**：⑪ 国际化/无障碍 · ⑮ 安装/更新/分发 · ⑯ 入门/引导/帮助 · （遥测/反馈并入⑦⑯）

**覆盖性判断**：原 12 领域表覆盖了交互/内核/数据/工程四层的多数，但漏了：
- ⑬ **规划/自主/多步执行**——Agent 与「聊天机器人」的本质区别（计划模式、TODO 分解、多步自主、反思纠错），原表只在 UI 里零散提到。
- ⑭ **与 Minecraft/HMCL 深度集成**——这是本应用的垂直灵魂（AI 操作 MC 的广度深度、场景覆盖率），通用客户端 taxonomy 里没有、但本项目必须单列。
- ⑮ **安装/更新/分发**——打包、自动更新、与上游 HMCL 的关系。
- ⑯ **入门/引导/帮助/文档**——用户能否上手（部分在 ①A14，独立成域）。

已把 ⑬⑭⑮⑯ 补入进度表。**最终判断（2026-07-02）：1–16 领域全部审计完成，taxonomy 已完全覆盖 → 满足结束标准，循环停止。** 后续以「跨领域高优先待办汇总」+ 各领域待办 + 任务 #53–#65 为施工依据。

# 安全机制

HMCL-AE 的 AI agent 能执行 shell 命令和文件操作,所以它有**真正的安全护栏**(不只是提示词告知)。

## 审批模式

在 **「AI 设置 → 工具」** 可以设置审批模式。有三种:

| 模式 | 行为 |
|---|---|
| **安全(Safe)** | 读/写/网络工具自动执行;**只有危险操作**(如 shell 危险命令、删除实例)需要用户确认 |
| **询问(Ask)** | 同安全,危险操作需要确认。未来可能增加更多细粒度控制 |
| **YOLO** | 所有工具都不需要确认,直接执行 |

> 注意:安全模式**不是**每次工具调用都确认(之前的版本是这样——已修正)。现在大多数操作自动通过,只有**危险操作**拦下来。

## 危险命令拦截

以下类型的 shell 命令会被标记为"危险",执行前弹窗让你确认:

- `rm -rf` / `rmdir /s` / `del /s`(删除文件/目录)
- `Remove-Item -Recurse`(PowerShell 递归删除)
- `format` / `mkfs`(格式化磁盘)
- `shutdown` / `reboot` / `halt`(关机/重启)
- `reg delete`(注册表删除)
- `killall -9`(强制杀进程)
- `chmod -R 000`(权限锁定)
- 明显是 fork 炸弹的模式
- ……

**你可以选择「否」拒绝执行**,此时 AI 会被告知"用户拒绝了,请换一种方式"。

## 工具权限分级

每个工具实现时都标注了权限等级:

| 权限 | 说明 |
|---|---|
| READ_ONLY | 读取、搜索、浏览 |
| CONTROLLED_WRITE | 写文件、装 mod |
| DANGEROUS_WRITE | shell 中的危险命令、删除实例 |
| EXTERNAL_NETWORK | web_search、web_fetch |

审批模式 + 权限等级 = policy check → 自动通过 / 拦下确认 / 拒绝。

## API Key 安全

- API Key 在 `.hmcl/ai-settings.json` 中**仅做 Base64 编码**,不加密。
- 确保你运行 HMCL-AE 的电脑是可信的。
- 不要把 `.hmcl/` 目录分享给他人。

## 已知的安全局限

- 危险命令列表是**模式匹配**——可能被绕过(如把 `rm` 拆成变量)。
- 一些中等危险操作(如覆盖配置文件)目前被归到 CONTROLLED_WRITE 不拦截。
- 安全机制在持续完善中,欢迎反馈。

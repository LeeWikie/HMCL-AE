# 已知问题

这是 beta 软件。以下诚实地列出了当前已知的问题和限制。

## 安装/下载是阻塞式的,没有进度条

让 AI 装版本/mod 时,下载期间聊天会"卡住"一阵,**没有任何进度显示**。下载时间取决于文件大小和网络速度,期间 UI 无反应。

**这是下一个要解决的重要问题**——把长任务接进 HMCL 原生的后台任务/进度系统。

## 不同模型的遵守程度不同

较强的模型(DeepSeek-R1、GPT-4o、Claude Opus 4.8)能更好地遵守提示词中的操作剧本;弱一些的模型可能会"忘记"先查版本列表、凭记忆猜版本,或忽略隔离状态。

## 会话编辑的动作条仅覆盖历史消息

刚发出的消息在切换/重载会话后才会出现下方图标动作条(复制/编辑/重发/分支/删除)。

## 部分操作缺乏专门的 UI

- 游戏设置(内存/JVM 参数/Java 版本/窗口大小)通过 `config-hmcl` skill 修改配置文件,**没有专门的可视化界面**。
- 账号登录(微软 OAuth / 离线 / authlib-injector)**尚未自动化**(工具未做)。

## 数据包管理未做

不支持搜索/安装数据包(datapacks)。

## 整合包导出未做

只有安装整合包,没有导出。

## 彩色 emoji

- 默认用系统渲染(黑白/可能缺失)。
- 彩色 emoji 需在设置里**手动开启 + 联网下载**(Noto Emoji)。

## 工具调用卡

- 流式回答中**实时**的工具调用卡不保留到会话重载后(重载只显示合并的文字。)

## 反馈

如果你遇到以上之外的问题,请在 [GitHub Issues](https://github.com/LeeWikie/HMCL-AE/issues) 提交,附上:
- 用的哪个模型
- 你做了什么操作
- 需要的日志(`.hmcl/logs/` 里的最新文件——现在每次工具调用都写日志了)

# 构建与贡献

## 从源码构建

### 环境要求
- **JDK 17+**(推荐 OpenJDK 17)
- **Gradle**(项目自带 wrapper,不需要全局安装)

### 构建步骤

```bash
# 克隆
git clone https://github.com/LeeWikie/HMCL-AE.git
cd HMCL-AE

# 编译(带测试)
./gradlew :HMCLAI:compileTestJava :HMCL:compileTestJava

# 仅编译主代码(不跑测试)
./gradlew :HMCLAI:compileJava :HMCL:compileJava

# 运行
./gradlew :HMCL:run

# 构建可分发的 jar 和 exe
./gradlew :HMCL:makeExecutables
```

构建产物在 `HMCL/build/libs/`:

| 文件 | 说明 |
|---|---|
| `HMCL-<version>.jar` | fat jar,`java -jar` 运行 |
| `HMCL-<version>.exe` | Windows 可执行文件(也是合法 jar) |
| `HMCL-<version>.sh` | Linux/macOS Shell 脚本头包裹 |

## 模块结构

| 模块 | 说明 |
|---|---|
| **HMCLAI** | AI 核心——agent 循环、工具接口、搜索、skills、LangChain4j 适配 |
| **HMCL** | 启动器主程序——UI(AI 聊天/设置)、HMCL 操作工具(复用安装/下载 API) |
| **HMCLCore** | 核心服务——游戏仓库、下载引擎、版本管理、崩溃分析 |
| **HMCLBoot** | 引导 / 启动入口 |

## 贡献代码

1. Fork 本仓库。
2. 在 `ai-feature` 分支上开专门分支。
3. 修改,确保 `./gradlew :HMCLAI:compileJava :HMCL:compileJava` 通过。
4. 提交 PR 到 `ai-feature`。

> 请不要把 AI 相关的工具/UI 放到 HMCLCore——AI 代码应保持在 HMCLAI 或 HMCL/ui/ai/ 内。
